# v1.1.0 Feature Design

Two independent features, each delivered as a separate PR.

---

## Feature 1: Always-Visible Progress Bar

### Goal

Show video progress at all times without requiring hover. The seek bar itself stays visible in a minimal collapsed state and expands into its normal position within the controls on hover.

### Design

**Current behavior:** The entire `.irc-controls` container (including the seek bar inside `.irc-top`) is hidden (`opacity: 0`) and slides in on hover via `*:hover > .irc-controls`.

**New behavior:** Split the controls into two visibility states:

- **Resting (no hover):** The seek bar is always visible as a thin line spanning the full width of the video, positioned at the top edge. No thumb, no padding, no other controls â€” just the gradient progress line. It remains non-interactive in this state (`pointer-events: none`) to avoid interfering with Instagram's UI.
- **Hover:** The full controls bar appears as before. The seek bar transitions from its resting position/size into its normal position within `.irc-top`. The bottom row (play, volume, time, speed) fades in.

**CSS approach:**

The seek bar is pulled out of the `.irc-controls` opacity/transform animation. Instead, `.irc-controls` always has `opacity: 1` but the `.irc-bottom` row handles its own fade. The seek bar transitions between two states:

- Resting: full width of wrapper, `top: 0`, `left: 0`, `right: 0`, height 2px, no thumb, no padding, reduced opacity (~0.7)
- Hover: normal position within `.irc-top`, height 3px, thumb appears on hover, full opacity

This can be achieved by keeping the current DOM structure but changing which elements participate in the hover transition:

- `.irc-controls` â€” always visible, `opacity: 1`, no transform. Acts as a positioning container.
- `.irc-bottom` â€” gets the existing fade/slide transition that `.irc-controls` currently has (`opacity: 0; transform: translateY(-6px);` â†’ visible on hover).
- `.irc-top` (seek row) â€” always visible, but transitions its padding/margins and the seek bar height between resting and hover states.
- On hover, `.irc-top` animates from its resting appearance (flush, thin, edge-to-edge) into its normal padded position below `.irc-bottom`.

**No new DOM elements needed.** The existing seek bar serves double duty.

### CSS changes (`content.css`)

- Remove opacity/transform animation from `.irc-controls`
- Move the fade/slide animation to `.irc-bottom`
- `.irc-top` always visible; transitions padding and seek bar height
- In resting state: `.irc-top` has no margin, no padding; seek bar is 2px tall, edge-to-edge
- On hover: `.irc-top` gets its current margin/padding; seek bar grows to 3px
- Seek bar thumb hidden in resting state (already the case), visible on hover of controls

### Files touched

- `content.css` â€” rework hover transitions (main change)
- No TS changes needed â€” the sync tick loop already updates the seek bar continuously

## Feature 2: Picture-in-Picture Button

### Goal

Add a PiP toggle button to the controls bar. Uses the native browser PiP API.

### Design

**Button placement:** Added to `.irc-bottom` row, after the speed button (rightmost position). Uses the same `.irc-btn` styling as play/mute buttons.

**Icon:** New `pip.svg` icon â€” a simple PiP glyph (rectangle with smaller rectangle in corner). Added to `icons/` and registered in `ICON` map. Also needs to be added to `web_accessible_resources` in `manifests/base.json`.

**Behavior:**

- Click toggles PiP: if `document.pictureInPictureElement === video`, exit PiP; otherwise, request PiP.
- Uses `video.requestPictureInPicture()` / `document.exitPictureInPicture()`.
- Listens for `enterpictureinpicture` and `leavepictureinpicture` events on the video to update button state (icon swap or visual indicator).
- No preference persistence â€” PiP is a transient state.

**Error handling:** `requestPictureInPicture()` can reject if the video has `disablePictureInPicture` set or if there's no user gesture. Wrap in try/catch, fail silently.

**DOM changes (`dom.ts`):**

- Create `pipBtn` with class `irc-btn irc-pip` and title `"Picture-in-Picture"`.
- Add to `.irc-bottom` row after the speed wrap.

**Event changes (`events.ts`):**

- Click handler on `pipBtn` to toggle PiP.
- `enterpictureinpicture` / `leavepictureinpicture` listeners on video to update icon.

**Icon changes (`icons.ts`):**

- Add `pip` and `pipActive` (or just `pip`) to `ICON` map.

**Types changes (`types.ts`):**

- Add `pipBtn: HTMLButtonElement` to `ControlElements`.

### Files touched

- `icons/pip.svg` â€” new icon file
- `manifests/base.json` â€” add `pip.svg` to `web_accessible_resources`
- `src/types.ts` â€” add `pipBtn` to `ControlElements`
- `src/icons.ts` â€” add `pip` to `ICON`
- `src/dom.ts` â€” create and place PiP button
- `src/events.ts` â€” PiP toggle + state listeners
- `content.css` â€” no changes expected (reuses `.irc-btn`)
